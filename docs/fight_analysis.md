# Fight Analysis Engine

The prediction API now returns a structured `analysis` object alongside the existing prediction fields.

## What It Uses

The analysis layer uses only data already available to the backend:

- predicted winner and model probability
- confidence label
- Elo values and Elo availability
- weight class match or cross-division warning
- fighter metadata such as stance, height, reach, record, and weight class when available
- style labels and comparison notes generated by the existing feature layer
- data quality and missing-data warnings

It does not invent missing stats.

## What It Does Not Claim

The analysis is a scenario-based fight preview, not a guarantee. Round and exchange sections use cautious language such as:

- "One likely version of the fight is..."
- "The model profile suggests..."
- "If the fight stays at range..."
- "The later rounds could become more volatile..."

The app should not say a fighter will definitely win, finish, or dominate.

## AI Provider Behavior

AI-generated summaries are disabled by default.

Default behavior:

```text
ENABLE_AI_SUMMARY=false
AI_SUMMARY_PROVIDER=none
```

With no AI provider enabled, public predictions still work and use the deterministic fallback summary. That fallback is richer than a canned phrase list because it builds sections from the actual matchup data, labels, warnings, and available signals.

## Optional Environment Variables

These are optional and not required for normal predictions:

```text
ENABLE_AI_SUMMARY=false
AI_SUMMARY_PROVIDER=none
AI_SUMMARY_MODEL=
AI_SUMMARY_TIMEOUT_SECONDS=10
```

No public prediction depends on Ollama, OpenAI, or any paid provider.

## Why Raw Signal Weights Are Hidden

The UI avoids displaying raw signal weights such as "Elo 35%" unless the app can explain exactly what that signal means, how it was measured, and why it mattered in the specific fight.

Instead, it shows qualitative drivers:

- "Elo profile favors Fighter A."
- "The model sees this as a close matchup."
- "Cross-division uncertainty lowers confidence."
- "Limited fighter data makes this prediction less certain."

This is clearer for users and avoids overstating model precision.
